## 2. 시나리오 (5~10줄)
이 프로그램은 파이썬 기초 문법을 활용해 대학생의 제한된 투자금을 체계적으로 관리하고 리스크를 줄이기 위한 '스마트 분할매수 시뮬레이터'입니다.

단순 수익률 확인을 넘어, 주가 하락 시 추가 매수(물타기)를 진행했을 때의 포트폴리오 변화를 미리 예측하는 실용적인 목적을 가집니다.
1차 버전(V1.0)에서는 사용자로부터 관심 종목명, 현재 보유 중인 매수 단가와 수량, 추가 매수 예정 단가와 수량을 차례대로 입력받습니다.
입력된 데이터를 바탕으로 기초적인 사칙연산과 형변환을 거쳐, 추가 매수 후의 '예상 평균 단가'와 '필요 총자금'을 즉시 계산해 출력합니다.
향후 2차 단계에서는 조건문을 도입하여 현재 주가에 따른 수익률을 계산하고, '익절/손절 구간 진입' 및 '비중 과다'를 진단하는 경고 시스템을 더할 것입니다.
3차 단계에서는 반복문(while)과 콘솔 메뉴를 적용하여, 프로그램을 끄지 않고도 다양한 분할매수 시나리오를 무한 반복하며 테스트할 수 있게 개선합니다.
마지막 4차 단계에서는 파이썬 리스트(배열)를 통해 여러 주식 종목을 하나의 종합 포트폴리오로 통합하여 관리할 계획입니다.
최종적으로 수익률 계산식을 함수로 모듈화하고, 파일 입출력을 통해 나만의 시뮬레이션 결과를 영구적으로 저장하고 불러오는 실전 프로그램으로 완성하고자 합니다.

## 🟦 [1차 과제: V1.0] 시나리오 기획
**✨V1.0 개발 내용:**
사용자로부터 '종목명(문자열)', '기존 매수 단가 및 수량(정수)', '추가 매수 예정 단가 및 수량(정수)'을 input() 함수로 차례대로 입력받습니다.
사칙연산(덧셈, 곱셈, 나눗셈)을 통해 '추가 매수 후 예상 평균 단가'와 '필요한 총 추가 투자금'을 계산하여 화면에 출력하는 기초 시뮬레이터를 구현했습니다.
조건문(if)이나 반복문(while) 없이 오직 변수 선언, 형변환(int(), float()), 그리고 f-string 출력 포맷팅만을 활용해 직관적인 시작 화면과 결과 화면을 구성했습니다.    

### **🤖 AI 파트너십 과정**

 1. **내용 1 : 컨셉 도출**
    * **프롬프트 요약:** "평소 관심 있는 주식 투자를 주제로, 기존 MTS 기능에 창의성을 더하되 대학생 수준에 맞는 실용적이고 논리적인 콘솔 프로그램 시나리오를 추천해 줘."
    * **적용 내용:** AI가 제안한 여러 아이디어 중 단순한 주식 계산기나 유치한 게임 요소를 배제하고, 제한된 투자금을 효율적으로 관리할 수 있는 **'스마트 분할매수(물타기) 시뮬레이션'** 이라는 수리적이고 분석적인 접근의 창의적 컨셉을 도출하여 적용했습니다.
      
 2. **내용 2 : 데이터 구조 및 로드맵 도출**
    * **프롬프트 요약:** "1차 과제 필수 요건(조건문/반복문 배제, 다양한 변수와 3가지 이상의 자료형 활용)을 충족하기 위한 주식 분할매수 시뮬레이터의 입출력 데이터 구조 모델링 논의"
    * **적용 내용:** 포괄적인 의미의 '투자 금액' 변수 하나만 사용하는 대신, 데이터 특성에 맞춰 문자열(str - 종목명), 정수형(int - 기존/추가 매수 단가 및 보유 수량), 실수형(float - 나눗셈을 통한 예상 평균 단가)으로 자료형을 명확히 분리하여 입력 데이터와 연산 결과의 구조적 완성도를 높임.


### **🛠️ Troubleshooting & 기술 회고:** 
**[에러 발생 및 해결]** 
input()으로 입력받은 기존 주식 수량과 추가 수량을 + 연산자로 더했을 때, 덧셈이 되지 않고 두 숫자가 문자열처럼 이어져서(예: 50+50=5050) 출력되는 논리 오류를 겪었습니다. 이를 통해 input()으로 받은 데이터는 기본적으로 문자열(str) 임을 깨닫고, int() 함수를 사용해 정수로 명시적 형변환을 거쳐 올바르게 사칙연산이 되도록 코드를 수정했습니다.

**[새롭게 배운 점]** 평균 단가를 구하기 위해 나눗셈(/) 연산을 진행하면 그 결과값이 항상 실수(float)로 반환된다는 파이썬의 특징을 새롭게 배웠습니다. 또한, 계산된 여러 변수를 화면에 출력할 때 f"{변수명}" 형태의 f-string을 활용하니 코드가 훨씬 직관적이었으며, 실수로 나온 평균 단가를 {avg_price:.1f}로 포맷팅하여 소수점 첫째 자리까지만 깔끔하게 출력되도록 제어하는 실무적인 방법을 익혔습니다.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**

  * 이번 과제는 "주식 포트폴리오 진단 및 투자 성향 분석 시스템" 구현으로 사용자로 부터 입력받은 종목데이터를 리스트에 구조화 하여 저장하고, 수익률 계산을 통해 련재 투자 상태를 정밀 전단하는 프로그램을 구축하는 것입니다.
  * 특히 if-elif-else 조건물을 활용해 수익률에 따른 투자 등급을 부여하고 and / or 논리 연산자를 이용하여 현금 비중과 하락 폭을 동시에 고려한 "물타기 권장" 등 지능형 투자 조언 로직을 완성했습니다
  * input() 으로 종목명, 매수 평단가, 보유수량 ,현재가를 입력받아 하나의 리스트(stock_info)에 저장.
  * 수익률 공식 적용 : ((현재가 - 평단가) / 평단가) * 100을 통해 소수점까지 포함된 수익률 산출.
  * if-elif-else를 사용하여 '적극 매도'부터 '강력 홀딩'까지 5단계 투자 진단 로직 추가.
  * and 연산자를 활용하여 '수익률 -15% 이하 이면서 여유 자금이 있는 경우'에만 '물타기 적기' 칭호를 부여하는 기능 구현.
    
### **🤖 AI 파트너십 과정**

1. **내용 1: 투자 진단 로직 및 등급 구간 도출**
    * **프롬프트 요약:** "주식 시뮬레이터에서 수익률에 따라 투자 등급을 5단계로 나누고 싶어. 대학생 수준에서 공감할 수 있는 등급 명칭과 논리 연산자를 섞은 추천 로직을 제안해 줘."
    * **적용 내용:** AI가 제안한 '유망주', '존버 필수', '탈출 권고' 등의 등급 명칭을 반영하고, 단순히 수익률만 보는 것이 아니라 and 연산자를 써서 하락 폭과 투자 비중을 동시에 체크하는 복합 조건 아이디어를 코드에 적용함.

2. **내용 2: 데이터 구조 최적화 자문**
    * **프롬프트 요약:** "여러 개의 변수를 따로 선언하는 것보다 리스트 하나에 담아서 관리하는 게 왜 효율적일까? 2차 과제 수준에서 리스트와 조건문을 결합해 확장할 수 있는 방향을 알려줘."
    * **적용 내용:** 개별 변수 대신 리스트 인덱싱을 활용해 데이터를 관리하는 구조를 확립함. 이를 통해 '기초 데이터 리스트화(2차) → 반복문을 통한 다종목 관리(3차) → 함수 기반 포트폴리오 모듈화(4차)'로 이어지는 단계별 확장 계획을 구체화함.
    
### **🛠️ Troubleshooting & 기술 회고:**

1. **문제 1: 리스트 인덱싱 접근 오류 (IndexError)**
    * **원인:** 리스트에 저장된 순서는 0번부터 시작하는데, 사람의 생각대로 1번부터 데이터를 불러오려다가 프로그램이 멈추는 에러가 발생함.
    * **해결:** 파이썬 리스트의 첫 번째 요소는 무조건 [0]번 인덱스라는 기본 원리를 다시 복습하고, stock_info[0]부터 차례대로 매칭하여 해결함.
2. **문제 2: 조건문이 의도치 않게 중복 실행되는 현상**
    * **원인:** 모든 조건을 if로만 작성해서, 한 조건이 만족되어도 아래쪽 if문들이 불필요하게 계속 검사되어 결과가 여러 개 출력됨.
    * **해결:** 상호 배타적인 조건일 때는 elif를 사용하여 앞선 조건이 맞으면 나머지는 건너뛰도록 구조를 변경하여 로직의 효율성을 높임.

## 🟨 [3차 과제: V3.0] 무한 루프와 메뉴 시스템 (반복문) - 향후 작성 예정
### **✨3차 과제 업데이트 내용:**
* 이번 과제는 기존의 단발성 주식 코드를 확장하여, 사용자가 종료를 선택하기 전까지 무한히 반복 실행되는
'메뉴 기반 주식 관리 프로그램' 으로 고도화 하는 것입니다.
* 기존 2차 과제의 기능들(입력, 연산, 출력 등)을 완전히 쪼개어 최소 3개 이상의 개별 함수(def)로 분리 정의하고, 메인 로직은 함수 호출 위주로 간결하게 구성했습니다.
* while True 무한 루프를 도입하여 사용자가 주도적으로 프로그램을 제어할 수 있는 메인 메뉴 시스템(1. 종목 입력, 2. 투자 진단, 3. 포트폴리오 마감, 4. 프로그램 종료)을 구축 했으며, 종료 선택 시에만 break를 통해 안전하게 프로그램이 끝나도록 구현했습니다.
* 함수 간의 유기적인 데이터 교환을 위해 매개변수와 반환값(return)을 적극적으로 활용하였고, 프로그램 전반에서 공유되는 주식 데이터를 관리하기 위해 전역 변수와 global 키워드를 적절히 매칭했습니다.

    
### **🤖 AI 파트너십 과정**
 1. **내용 1** 효율적인 메뉴 구조와 함수 분리 설계 질문
    
    * **프롬프트 요약:** "기존 주식 평단가 계산 코드를 무한 루프 메인 메뉴 형태로 바꾸고 싶어. 메인 코드가 너무 길어지거나 복잡해지지 않게 함수 3개로 깔끔하게 쪼개는 구조와 흐름을 추천해 줘."
    
    * **적용 내용:** AI가 제안한 컴포넌트화 개념을 수용하여 메인 루프 내부는 단순 함수 호출로만 채우고, 데이터 입력(input_stock), 수익률 계산(calc_profit), 결과 출력(print_result) 기능으로 깔끔하게 역할을 분담시켜 코드의 구조적 가독성을 극대화함.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 반환값(return)을 받아왔는데 메인 메뉴에서 활용하지 못하고 사라지는 현상

     * **원인:** 연산 함수에서 return profit_rate로 값을 열심히 돌려주었으나, 정작 메인 로직 함수 호출부에서 이 값을 담아줄 변수를 지정하지 않아 데이터가 공중분해됨.

     * **해결:** 함수의 반환값은 그냥 호출만 하면 안 되고 반드시 변수에 새로 대입하거나
 (result = calc_profit()), 다른 함수의 인자로 직접 넘겨주어야 데이터 흐름이 이어진다는 원리를 학습하고 코드를 수정함.

### 🟥 [4차 과제: V4.0] 모듈화 및 데이터 확장 (배열과 함수) - 🌟최종 완성 -- 향후 작성 예정
### **✨4차 과제 업데이트 내용:**
  * 내용 :  이번 과제는 단일 종목만 관리하던 기존 주식 진단 시스템을 다중 종목 누적 관리가 가능한 V3.0 시스템으로 확장하는 것입니다. 프로그램 종료 시 데이터가 휘발되는 문제를 해결하기 위해 파일 입출력 기능을 도입하였고, 사용자 입력 및 파일 로드 과정에서 발생할 수 있는 프로그램 강제 종료 예외를 제어하는 안정성 확보를 목표로 합니다.

  * 다중 자산 관리를 위한 이중 리스트(2D List) 구조 정의 및 append를 이용한 동적 데이터 누적 처리.

  * for 반복문과 리스트 인덱싱을 활용하여 중첩 데이터 구조를 깔끔한 격자 표 형태로 출력하는 모듈 구현.

  * open() 함수와 with open 구문을 활용하여 데이터를 텍스트 파일(portfolio_data.txt)에 내보내는 영구 저장 시스템 구축.

  * 예기치 못한 입력 오류(ValueError) 및 참조 파일 누락(FileNotFoundError) 상황에 대응하는 최소 2곳 이상의 try-except 예외 처리 구현.

    
### **🤖 AI 파트너십 과정**
 1. **내용 1:**  단일 리스트에서 이중 리스트 데이터 구조로의 확장 자문
    * **프롬프트 요약:**  3차 과제에서는 종목 하나를 portfolio_info라는 리스트에 저장했는데, 이번엔 사용자가 메뉴를 반복하면서 여러 개의 주식 종목을 계속 누적해서 저장해야 해. 아직 딕셔너리를 배우지 않은 기초 문법 단계에서, 리스트만 중첩해서 여러 종목을 관리하고 이걸 표 형태로 깔끔하게 꺼내 쓰는 가장 직관적인 제어 로직을 가르쳐줘."
    * **적용 내용:** AI가 제안한 전역 변수 기반의 텅 빈 대형 리스트(portfolio_list)를 먼저 선언하고, 1번 메뉴가 실행될 때마다 개별 종목 리스트를 생성해 리스트 내부의 요소로 append하는 방식을 채택했습니다. 데이터를 꺼낼 때는 바깥 리스트를 for문으로 순회하며 내부 요소들을 인덱스([0], [1] 등)로 호출하여 완전한 다차원 배열 제어 방식을 코드에 이식했습니다.
      
 2. **내용 2:**  텍스트 파일 저장 방식 및 복원 과정에서의 형변환 예외 처리 조언
    * **프롬프트 요약:**"이중 리스트에 담긴 데이터를 텍스트 파일로 저장했다가 다시 프로그램으로 읽어오려고 해. 그런데 외부 라이브러리 없이 콤마(,)로 split해서 데이터를 자르다 보니, 읽어온 데이터가 전부 문자열(str)로 인식되어서 기존의 수익률 계산 함수나 숫자 포맷팅 출력에서 에러가 터져. 이걸 비정상 종료 없이 안전하게 숫자로 되돌리고 파일이 없을 때의 에러도 막는 방안을 알려줘."
    * **적용 내용:** AI의 피드백을 바탕으로, 파일에서 문자열 데이터를 한 줄씩 읽어올 때 split()을 적용한 후 각 인덱스의 데이터를 float()과 int()로 일일이 다시 재형변환(Type Casting)해 주는 파싱 단계를 추가했습니다. 이와 동시에 파일이 실재하지 않을 때 터지는 치명적인 시스템 예외를 방어하기 위해 read 블록 전체를 try-except 구조로 감싸는 안정화 기법을 적용했습니다.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 사용자가 매수 가격이나 수량 입력란에 '원' 또는 '주'라는 단위를 함께 입력하거나 실수로 문자를 입력했을 때 프로그램이 완전히 뻗어버리는 현상 발생.
     * **원인:** input()으로 받은 문자열 변수를 float()이나 int()로 강제 형변환하는 과정에서, 숫자가 아닌 일반 문자형이 섞여 들어오자 파이썬 엔진이 변환 불가능 판정을 내리고 ValueError 예외를 발생시켰기 때문입니다.
     * **해결:** 1번 메뉴의 사용자 입력 로직 전체를 try-except ValueError 구문으로 감싸주었습니다. 문자가 입력되어 에러가 감지되면 시스템이 다운되는 대신 에러 문구를 정중하게 출력하고 return을 통해 잘못된 데이터가 이중 리스트에 섞이지 않도록 차단하여 안정성을 확보했습니다.
